web压测工具http_load

发表于 2014-10-20   |  

http_load是一款linux上的web压测工具，能够以并行方式批量发出http请求，用以测试web服务器的吞吐能力。然而跟其他压测工具不一样，http_load以单进程运行，所以通常不会将客户端机器搞挂。
http_load同样支持https压测。详细介绍和下载请见官网http://acme.com/software/http_load/

##参数
支持以下参数（支持使用参数首字母简写，例如rate简写成r）

  • -rate N : 发送频率（每秒发起几次请求）。
  • -jitter : 只能与rate同时使用。表示实际的访问频率会在rate设置的值上下随机波动10%的幅度。
  • -parallel N : 并行请求数量。（rate和parallel参数至少指定一个，且最终结果不严格保证符合预期。）
  • -second N : 请求持续时间。
  • -fetches N : 请求次数。second和fetches参数至少指定一个。
  • -verbose : 每分钟输出当前测试进度报告。
  • -timeout secs : 超时时间，默认60s。测试报告会显示超时次数。
  • -checksum : 是否校验结果。如果指定了这个参数，第一次访问某个url时会记录下返回结果，下一次再请求时如果与第一次不一致就会报警。
  • -throttle : 限流模式。限制网速到33.6Kbps，用以模拟调制解调器时代的网速。
  • -proxy HOST:PORT : 通过web代理发送请求。
  • -sip sip_file : 指定一个source ip文件，每行一个ip，用于随机指定请求的来源地址。这个参数能让服务器误以为测试请求来自多台机器。
  • url_file : 提供一个url列表，每行一个，每次请求都是随机抽取url。
