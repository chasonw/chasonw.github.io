快速搭建hexo博客

发表于 2014-08-21   |  

前阵子费了不少功夫终于把wordpress搭建起来，这阵子又接触学习了hexo博客系统，相见恨晚，才知道一个博客可以分分钟搭建起来。
hexo介绍见官网http://hexo.io，号称是基于Node.js开发的快速、简单、威猛的博客框架。
什么样的人会对hexo感兴趣呢？或者说必须要达到什么条件才有办法使用hexo呢？

  • 喜欢写文字，记博客
  • 喜欢markdown语法（不喜欢word或者富文本编辑器复杂难以维护的格式）
  • 技术控，最好有过web开发相关经验（如果有人帮你搭建，可以忽略这条）
  • 希望博客自适应移动端

快速搭建

首先你要有台机器（mac/linux/windows都行，首推mac），接着：

  1. 安装Node.js[^nodejs]
    • 到官网http://www.nodejs.org/download/下载安装（不要担心，下载之后一直下一步即可）
    • node -v验证node是否正常安装，
    • npm -v验证npm是否正常安装。npm是Node.js的包管理器。
  2. 安装hexo：npm install hexo -g，安装完hexo version验证
  3. 创建博客目录：hexo init /server/hexo-blog，cd /server/hexo-blog后可以看到生成一系列文件
  4. 安装hexo依赖包：npm install
  5. 启动服务：hexo server，会提示博客地址http://localhost:4000/，访问即可。


怎么样？简单地5步几条命令，没有隐藏的旁支操作，童叟无欺！

对比搭建wordpress

再回忆下当初在Ubuntu上搭建wordpress的流程：

  1. 安装webserver
  2. 安装php
  3. 安装mysql
  4. 配置nginx服务
  5. 下载wordpress源码
  6. 配置wordpress：例如db配置
  7. web界面安装wordpress
  8. 启动php-fpm，nginx

看起来也没几步是把？too naive，每一步的操作远比一条命令复杂，而且还还很容易出错要解决。

博客功能

hexo跟Node.js一样，是个新生儿，相比wordpress功能没有那么齐全，但博客的基本功能都提供了。麻雀虽小五脏俱全。

  • 写博客
  • 自定义主题
  • 独立页面
  • 分类
  • 标签
  • SEO，例如meta、自定义链接
  • 评论（需要安装插件）

写文章

hexo并没有使用db存储数据，而是使用markdown的语法文件存储。写文章很简单：

  1. 到根目录，hexo new my-first-blog会在source/_posts目录生成一个markdown文件my-first-blog.md
  2. 刷新博客首页会发现多出了一篇文章《my-first-blog》，点进去是空内容。
  3. 编辑md文件，用markdown语法写文章即可（随时保存，刷新页面就会发现博客已经自动更新了）

如果不清楚markdown语法，可以自行google之。有时间博主会写一篇markdown的介绍。

部署

两种方式部署博客：

  • 静态化页面部署，也就是生成静态html页面，随便部署到哪个服务器下都可以访问。
    1. 执行hexo generate，会在public目录下生成一些静态文件。
    2. 可以本地直接打开访问，或者通过ftp等方式上传到服务器上访问。
  • 动态server部署
    1. 到服务器上搭建下hexo
    2. 更改_config.yml的port为80，server_ip为服务器ip或者域名
    3. hexo server开启服务，访问即可。

博客内容通常不会变化太频繁，因此建议使用静态化页面部署即可，服务器基本上不需要什么负载。
而本地环境可以使用server部署来写博客，写完之后静态化再上传发布。

github pages

本章节需要你了解点github和git的知识，否则可能比较难以进行。

github pages 提供了放置静态网页文件的功能，通过github来部署文件。因此可以利用github pages来部署博客的目的。现在我们就把刚刚创建的hexo-blog上传到pages上。

快速命令

hexo init /server/hexo-blog  #创建博客
hexo new draft <title>       #创建新草稿
hexo publish [layout] <filename> #发布草稿（相当于new draft + publish相当于 hexo new）
hexo new <title>             #创建新文章
hexo server                  #启动服务
hexo server --drafts         #启动服务（带草稿）
hexo generate                #生成静态文件
hexo generate --deploy       #生成静态文件并且部署
#hexo
